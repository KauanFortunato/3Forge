# Export Runner

Minimal playground to validate the TypeScript files generated by the 3Forge editor.

## Workflow

1. Export the component as TypeScript from the editor.
2. Save one or more generated files at:

```text
playgrounds/export-runner/src/generated/*.ts
```

3. Start the runner:

```bash
npm run dev:export-runner
```

4. Choose the file in `Generated File`.
5. Click `Build export`.

If the generated class exposes animation methods such as `play`, `pause`, `stop`, `seek`, or `playClip`, the runner enables the matching controls automatically.

## What the runner does

- detects the exported class automatically
- instantiates it with a JSON options object
- calls `build()`
- mounts `group` into a simple Three.js scene
- frames the result in a viewport with orbit controls
- lets you call runtime animation methods when present

## Scope

This runner is intentionally separate from the main editor shell. It exists only to validate runtime exports and keep that workflow simple and isolated.
