# Funcionalidades

Esta pasta concentra o registo funcional do `3Forge`.

## Regra de trabalho

Sempre que uma funcionalidade nova for adicionada ao projeto:

- ela deve ser registada nesta pasta, e nao em `help/`
- a descricao deve focar no comportamento entregue ao utilizador
- quando fizer sentido, a entrada deve apontar para os arquivos principais alterados
- se a funcionalidade exigir cobertura nova, os testes relacionados devem ser citados

## Estrutura

- `README.md`: define a convencao desta pasta
- `HISTORICO.md`: historico interno das mudancas funcionais adicionadas, alteradas ou removidas

## Como usar

Para funcionalidades novas, criar ou atualizar um documento nesta pasta com:

- nome da funcionalidade
- objetivo
- comportamento principal
- limites ou observacoes relevantes
- testes cobrindo a entrega, quando existirem

O `HISTORICO.md` nao e um changelog de producao. Ele serve como registo interno de evolucao do projeto durante o desenvolvimento.
